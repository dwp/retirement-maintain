Iteration 3

burnley research
Date of design
March 2020
Designers
Emily Wilkinson and Colin Jones

Scenario

  • Agents only see one task at a time - the next highest priority as determined by the back end
  • New navigation design allowing agent to go between the task they’re in, to changes and enquiries or new claims journeys
  • Pattern in this scenario allows an agent to confirm they’ve completed a manual task

Business requirements

  • Agents should only be able to work on one task at a time
  • Agent needs to see what the task is before continuing into it to action it
  • There needs to be a way of ending the task once complete so that it doesn’t appear back in the queue

Research notes

  • Agents found the task straightforward and simple to complete
  • Some agents said that agents do cherry pick tasks if they see a list of them, this way they can't do that
  • Some agents said a list of tasks is often stressful to see, this design takes away that stress
