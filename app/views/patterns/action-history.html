Action history

Timeline

The basic timeline events are used across the service, the timeline shown depends on what area of the service the agent is in. Personal, contact, award and payment all have individual timelines with events related to the information being shown. 

How to use it
Title
What has happened: taken from a standard list of events. Each title should have only one event, for example: Date of death added, Date of death verified.
For usability titles should be between 3 and 8 words long, if you need to add more context then use the 'Details' feature.

Byline
The byline tells a user whether the action was completed by an Agent, the System or Online (by the claimant).
This is useful if more than one person can work on a case, or if your timeline includes both agent actions and automated events.

Description
Not used in GySP. If this detail is needed then consider using the 'Details' feature.
It is important for the usability of the timeline that events are simple and consistent in their appearance.

Details link
An event can include a text link to more detailed information. This link does not reveal content in the timeline itself; instead it takes the user to the information in another page or interface.
This maintains the position of other timeline items and avoids overcrowding the interface with additional detail.
You will need to use visually hidden text on the links so they are accessible.

Date
The date the action was completed.

Full history

A chronological view of all events since the State Pension was claimed using information from the timelines in the Personal, Contact, Award and Payment tabs.

Future iterations
We need to consider how we can help users understand that a notification was sent off of the back of an action. In some tasks a separate 'X letter sent' event is created in the timeline.

This view would allow agents to quickly see the notifications are. The plan is to then have a digital version of notifications for an agent to view.

Event details

Timeline details links are used to get more information about a timeline event. The timeline shows an event and the day it was recorded on the system, in some cases events span over a period of time and there is a user need to be able to see these dates. There is also a need to be able to see more details about an event to gain a full understanding of what has happened, for example the fraud event shown. 

Some areas of the service have their own timeline relevant to the lower level of detail. Such as payment details and living and working abroad details.

Help with nationality
We need to know your nationality so we can work out which elections you’re entitled to vote in. If you cannot provide your nationality, you’ll have to send copies of identity documents through the post.
